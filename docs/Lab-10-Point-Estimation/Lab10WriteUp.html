1.1 Altham’s Multiplicative Binomial Distribution

The PMF for Altham’s Multiplicative Binomial Distribution is \[f_X(x) = \frac{1}{c} {n \choose x} p^x (1-p)^{n-x} \theta^{x(n-x)}I(x \in \{0, 1, ..., n\})\] where \[c = \sum_{i=0}^n {n \choose i} p^i (1-p)^{n-i} \theta^{i(n-i)}.\] Just so we start on the same page, I have included my R function for computing probability using this distribution.

Code 
daltbinom <- function(x, size, prob, theta){
  normalizing.constant <- 0
  for(j in 0:size){
    normalizing.constant <- normalizing.constant + 
      choose(size, j) * prob^j * (1-prob)^(size-j) * theta^(j*(size-j))
  }
  # Probability Mass at x=x
  (1/normalizing.constant) *
    choose(size, x) * prob^x * (1-prob)^(size-x) * theta^(x*(size-x)) *
      (x>=0)*(x<=size)
}

2 Altham’s Multiplicative Beta Binomial Distribution

The PMF for Altham’s Multiplicative Beta Binomial Distribution is \[f_X(x) = \left[\int_{0}^1 \left(\frac{1}{c} {n \choose x} p^x (1-p)^{n-x} \theta^{x(n-x)}\right)\left(\frac{\Gamma(\alpha+\beta)}{\Gamma(\alpha)\Gamma(\beta)} p^{\alpha-1}(1-p)^{\beta-1}\right) dp\right]I(x \in \{0, 1, ..., n\})\]

Just so we start on the same page, I have included my R function for computing probability using this distribution.

Code 
daltbbinom.single <- function(x, size, alpha, beta, theta){
    integrand <- function(p) {
    sapply(p, # integrate will pass in a vector, so we use sapply to apply the following
           function(p_val){
             # f(x|p) * f(p|alpha, beta)
             daltbinom(x, size, p_val, theta) * dbeta(p_val, alpha, beta)
           }
    )
  }
  # Probability Mass at x=x
  # integrate(integrand, lower = 0, upper = 1)$value * (x>=0)*(x<=size)
  # I shrink the bounds below to avoid log(0)=-infinty
  # I use try() to catch when there is an error and avoid crashing
  res <- try(integrate(integrand, lower = 1e-7, upper = 1 - 1e-7)$value * 
                  (x>=0)*(x<=size), 
             silent = TRUE)
  # If there is an error, return a near-zero probability
  if(inherits(res, "try-error")) return(0.1^6)
  
  res
}
# Write a function that works on a vector
daltbbinom <- function(x, size, alpha, beta, theta){
  sapply(X=x, FUN=daltbbinom.single, size=size, alpha=alpha, beta=beta, theta=theta)
}

exk = function(k, size, prob, theta){
              c = sum(sapply(0:size, function(x){
              choose(size, x) * prob^x * (1-prob)^(size-x) * theta^(x*(size-x))}))
  

              numerator = sum(sapply(0:size, function(x){
              x^k * choose(size, x) * prob^x * (1-prob)^(size-x) * theta^(x*(size-x))}))
  
              numerator / c
}

4.3 Part c

Write a function that computes the log-likelihood for Altham’s Multiplicative Binomial Distribution, given data (x) and proposed parameter values \(p\) (prob) and \(\theta\) (theta). Solution:

Code 
loglike_altbinom = function(x, n, prob, theta){
  c = sum(sapply(0:n, function(i){
          choose(n, i) * prob^i * (1 - prob)^(n - i) * theta^(i * (n - i))}))
  
  loglike = sum(sapply(x, function(xi){
      log(choose(n, xi)) + xi * log(prob) + (n - xi) * log(1 - prob) +
      xi * (n - xi) * log(theta) - log(c)}))
  
  return(loglike)
}

Solution: We can use the log-likelihood funciton to find maximums to \(p\) and \(theta\) by plugging in different values of \(p\) and \(theta\) until it returns a high enough value, or the peak of the funciton. This is the value combination that shows the highest likelihood of occur with the set \(p\) and \(theta\).

total_legs = function(player, data){
        winner_legs = data |> 
        filter(Winner == player) |> 
        pull(WinnerLegs)
        loser_legs = data |> 
        filter(Loser == player) |> 
        pull(LoserLegs)

}

5.4 Executive Summary

Summarize the work you’ve done here to provide a brief (200-300 word) data-driven summary of the 2024 Premier League season using your work in Lab 10. Your summary should be professional, clear, and all claims should be corroborated with statistical support.

Solution: Although the premier league only has 8 contestants, there is a large gap between the top players and bottom players. Even with this gap, losing players still won an average of approximately 3.56 legs, meaning that most matches were competitive rather than a blow-out. The number of legs won isn’t as skewed as the win count between top and bottom which supports that the games were “close” at the least.

Altham’s Multiplicative Binomial model showed maximum likelihood estimates that higher-ranked players tend to have larger values of \(p\), indicating a higher probability of winning singular legs. Those players generally exhibit lower values of \(theta\), which indicates their more consistent performance. In contrast, lower ranked players have smaller \(p\) values and often higher theta which reflectes lower success rates and greater variability, though the difference isn’t substantial.

The Multiplicative Beta Binomial model adds the variability in leg winning ability compared to the Altham’s model. Players with higher estimated Beta means align with the top of the rankings, which confirmes stronger average performance. Larger Beta standard deviations and higher \(theta\) values among lower ranked players indicate more inconsistency.

Overall, the modeling supports the standings pf the leader board stronger players consistently win more legs and exhibit more stable performance, while weaker players win fewer legs and display greater variability, showing a clear skill gap across the league.
